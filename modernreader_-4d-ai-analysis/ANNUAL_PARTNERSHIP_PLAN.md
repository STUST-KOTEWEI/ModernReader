# ModernReader 年度合作與贊助計畫（供 Vendor / Sponsor 參考）

版本: v1.1 - 務實版  
最後更新: 2025-10-21

---

## 1. 簡介

ModernReader 是一個 4D 沉浸式 AI 閱讀與知識分析平台，結合自然語言理解（NLP/RAG）、語音（TTS/STT）、影像生成、穿戴式感測與情境運算。平台核心價值在於「讓內容依使用者的狀態自適應」，提供高專注、高回想、高樂趣的閱讀體驗。
本文件提供年度合作與贊助方案，協助硬體廠商（Vendor）與策略贊助商（Sponsor）快速了解合作模式、里程碑、技術整合與資安規範。

目前人力規模：1 人（目前為南臺科大學生）；預計於 11 月第 3 週加入 NCKU MI2S 與南臺實驗室，啟動跨校合作與研究。

## 2. 年度願景與 KPI（2025-2026 階段性目標）

### 2025 下學期目標（大一下，2-6月）

- 技術驗證：完成 WebXR Beta、Vendor SDK v1 GA
- 測試用戶：累積 50-100 位早期測試者（校內同學、合作夥伴）
- 內容合作：洽談 1-2 家出版社/平台進行 PoC 測試
- 穿戴整合：完成 Apple Watch 基礎整合（心率、HRV）
- 安全基線：完成 HTTPS 部署、基礎資安檢核（自我檢視清單）

### 2026 年度目標（大二，逐季達成）

- Q1-Q2：使用者成長至 200-500 人（校內推廣 + 早期採用者）
- Q2-Q3：日均閱讀時長達 15-20 分鐘/人
- Q3-Q4：穿戴連接滲透率達 10-15%（活躍用戶中）
- 全年度：與 1-2 家出版社完成內容上架測試
- 全年度：與 1 家穿戴裝置廠商完成深度整合（Apple Watch 或合作品牌）
- 持續性：維持 0 重大資安事故、通過校內資安規範審查

## 4. 合作模式

### 4.1 硬體廠商 Vendor（穿戴/耳機/感測器/AR 眼鏡）

- 整合方式：
  - Web Bluetooth / WebUSB / Web Serial（瀏覽器原生）
  - Companion App + WebSocket（行動端橋接）
  - 雲端 API（資料彙整與事件推播）
- 合作權益：
  - 於 MR 內建「Vendor Slot」掛載自家 Widget（見 index.html data-slot）
  - 聯名行銷版位、共同發表技術白皮書
  - 針對 Premium 用戶提供專屬體驗
- 技術標準：
  - 事件模型（JSON）：心率、HRV、步數、睡眠、壓力、電量、手勢
  - 時間同步：NTP/本地時鐘校正 ±50ms
  - 可靠度：> 99.9% 連線可用性（以月計）

### 4.2 贊助商 Sponsor（品牌/內容/學研）

- 贊助形式：
  - 品牌贊助（Logo / 專區 / 聯名活動）
  - 內容贊助（專題策展 / 聽讀課程 / 校園計畫）
  - 研究贊助（生理訊號與閱讀成效長期追蹤）
- 成效交付：
  - KPI 報表（曝光、互動、轉換）
  - 用戶回饋與問卷洞察（匿名化）
  - 技術/研究報告（可共掛單位 Logo）

### 4.3 學研合作（NCKU MI2S / 南臺等）

- 合作流程：
  - 前期文件：MOU/NDA 雙方簽署，資料使用與保密約定
  - 研究倫理：如涉及人體/生理數據，依學校規範辦理 IRB 或等同審查
  - 專案啟動：建立聯絡窗口、週會節奏、里程碑與交付品清單
  - 安全稽核：資料交換白名單、加密、存取權限最小化

- 資料與倫理：
  - 資料治理：去識別化、最小化蒐集、留存週期與刪除機制
  - 共享原則：僅供約定研究目的，不得二次利用；對外發表需雙方審閱

- 交付與里程碑（範例）：
  - M1：PoC 設計審查、資料欄位定義、IRB 提交
  - M2：資料收集開始、原型驗證、期中檢討
  - M3：研究結果彙整、論文/白皮書草案、對外發表計畫

- 聯絡窗口：
  - 學校端：指導教授/實驗室窗口（Email/Teams/Line）

## 5. 技術整合規格（摘要）

- 通訊協定：HTTPS + WSS（TLS1.2+），本地測試支援 `mkcert` 自簽憑證
- 資料格式：JSON（UTF-8），必要欄位含 `type`、`payload`、`timestamp`

- 事件範例：

```json
{
  "type": "biometrics",
  "payload": {
    "heartRate": 78,
    "hrv": 42,
    "sleepQuality": 86,
    "stressLevel": 28,
    "battery": 73
  },
  "timestamp": 1739798400000
}
```

- 前端掛載點：`#vendor-wearable-slot[data-slot="wearable-partner"]`
- 安全限制：CSP、來源白名單、跨網域 Cookie 禁用、不得蒐集個資

## 附錄 A：研究數據欄位與匿名化範本（依 IRB/校內規範為準）

本附錄提供與學研單位協作時可參考的資料欄位建議與匿名化原則，實際以 IRB 與各校規範為最高優先。

### A.1 欄位建議（最低必要原則）

- 基本欄位：
  - participantId（匿名代碼；不可回推個人身分）
  - sessionId（當次實驗/閱讀工作階段代碼）
  - createdAt（ISO8601 時間；必要時以日/時段級粒度儲存）
  - consentVersion（同意書版本）

- 裝置/環境：
  - device.model（如 Apple Watch 型號、手機型號）
  - device.os（OS 名稱/版本）
  - app.version / web.version（App 或 Web 前端版本）

- 生理摘要（如研究需要，且經 IRB 覆核）：
  - biometrics.heartRate（bpm；摘要或範圍值）
  - biometrics.hrv（RMSSD/SDNN 等摘要）
  - biometrics.stressLevel（模型輸出之區間/等級）
  - biometrics.sleepQuality（若為長期研究之彙整指標）

- 閱讀互動：
  - reading.contentId（內容代碼；以雜湊/假名化方式保存）
  - reading.contentType（文章/教案/聽讀等）
  - reading.durationSec（有效閱讀時長；排除長時間無互動）
  - reading.focusScore（專注度指標；經設計之歸一化分數）
  - reading.difficulty（自評或標註之難度等級）
  - reading.comprehensionScore（理解度測驗；如有）
  - events[]（時間序列事件；必要時以彙整/抽樣方式提供）

### A.2 匿名化與治理原則

- 不收集：姓名、學號、電話、Email、精確地理座標、完整 IP 等可識別資訊。
- 準識別資料（如 IP）：
  - 僅存不可逆雜湊值（加鹽）；不可儲存原始 IP。
  - 時間戳可能降精度（如只保留日/時段），降低重新識別風險。
- 假名化：對 contentId/participantId 使用隨機字串或雜湊；密鑰分離保存。
- 最低必要：依研究目的擇必要欄位；避免儲存原始連續生理訊號，優先儲存經統計之摘要指標。
- 留存與刪除：訂定留存週期（例如 90–180 天），期滿自動刪除或不可逆匿名化；保留稽核記錄。
- 存取控管：最小化權限；以角色權限與存取稽核紀錄控管資料查閱。
- 加密：傳輸與靜態加密（TLS/At-Rest Encryption）；金鑰管理與輪替機制。
- 對外發表：研究報告/論文對外前需雙方審閱；任何圖表/數據不得可識別個人。
